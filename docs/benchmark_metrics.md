# Alignment Quality Benchmark Metrics

This document defines the metrics computed by the alignment benchmarking script (`scripts/benchmark_alignment.py`) to evaluate the structural quality of manuscript collations generated by Collatex.

## Terminology

* **Witness**: A specific manuscript, copy, or source text instance being compared (e.g. `ebe_346`, `athous-iviron-450`).
* **Gap**: A placeholder or empty slot (represented as `[]` in Collatex JSON) in a column, indicating that the witness does not have a word at that alignment position (due to an omission in this witness or an addition in another).
* **Normalization Key (`n`)**: The comparative token string used under the hood by Collatex to determine matching and alignment (e.g., lemma, lemma+pos, or raw word).
* **Display Text (`t`)**: The original text token from the manuscript transcription. This is the text displayed on the Stemmaweb interface, regardless of which normalization key was used for the alignment.

## 1. Structural Collation Metrics

### Total Columns
* **Definition**: The total number of columns in the collation alignment table.
* **Interpretation**: In the Gothenburg Model of text collation, text is aligned as a directed acyclic graph (DAG). The columns represent aligned states. A smaller number of total columns generally indicates a more collapsed, successfully aligned graph where matching words across witnesses have been successfully merged into single columns.

### Consensus Columns
* **Definition**: The number of columns where all witnesses have a token and the comparison keys (normalizations) of those tokens are 100% identical.
* **Calculation**:
  $$\text{Consensus} = \sum_{c \in \text{Columns}} [(\text{gaps}_c == 0) \land (|\{n_{w,c} \mid w \in \text{Witnesses}\}| == 1)]$$
  where $n_{w,c}$ is the normalization value of witness $w$ at column $c$.

### Variant Columns
* **Definition**: Columns where witnesses exhibit textual differences. This includes:
  1. Columns without gaps where at least two witnesses have different normalizations.
  2. Columns with gaps where at least two of the witnesses that *do* contain a token have different normalizations.
* **Calculation**:
  $$\text{Variant} = \sum_{c \in \text{Columns}} [|\{n_{w,c} \mid w \in \text{Witnesses} \land \text{cell}_{w,c} \neq \emptyset\}| > 1]$$

### Gap Columns
* **Definition**: The number of columns in the alignment table where at least one witness has a gap (no token aligned at this position).
* **Calculation**:
  $$\text{Gap Columns} = \sum_{c \in \text{Columns}} [\exists w \in \text{Witnesses} \text{ s.t. } \text{cell}_{w,c} == \emptyset]$$

### Total Gaps
* **Definition**: The sum of all empty cells across the entire collation table.
* **Calculation**:
  $$\text{Total Gaps} = \sum_{c \in \text{Columns}} \sum_{w \in \text{Witnesses}} [\text{cell}_{w,c} == \emptyset]$$

### Alignment Density (Compactness)
* **Definition**: The proportion of non-empty cells in the collation table relative to the total possible cells (columns multiplied by witnesses).
* **Calculation**:
  $$\text{Density} = \frac{\text{Total Cells} - \text{Total Gaps}}{\text{Total Cells}}$$
  where $\text{Total Cells} = \text{Total Columns} \times |\text{Witnesses}|$.
* **Format**: Represented as a decimal between `0.0` and `1.0` (or `0%` to `100%`).

## 2. Performance Metrics

### Tokenization Time
* **Definition**: The CPU time (in seconds) required to preprocess the source XML-TEI files, run the NLP parser (CLTK/Stanza), perform lemmatization and Part-of-Speech tagging, filter deleted tags, and format the tokens.
* **Precision**: Rounded to 2 decimal places.

### Collation Time
* **Definition**: The round-trip HTTP time (in seconds) required to serialize the tokenized witness data, transmit the payload to the Collatex REST API, run the collation algorithm, and receive the aligned JSON response.
* **Precision**: Rounded to 2 decimal places.

### Total Time
* **Definition**: The overall execution time (in seconds) representing the sum of tokenization and collation phases.
* **Precision**: Rounded to 2 decimal places.
