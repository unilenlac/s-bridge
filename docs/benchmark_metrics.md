# Alignment Quality Benchmark Metrics

This document defines the metrics computed by the variant graph analysis (`scripts/analyze_variant_graphs.py`) and the alignment benchmarking script (`scripts/benchmark_alignment.py`) to evaluate the structural quality of manuscript collations generated by Collatex.

## Terminology

* **Witness**: A specific manuscript, copy, or source text instance being compared (e.g. `ebe_346`, `athous-iviron-450`).
* **Variant Graph / Collation Graph**: A Directed Acyclic Graph (DAG) representing the text collation. Nodes are reading/token units, and directed edges are transitions representing witness sequences.
* **Gap**: A placeholder or empty slot (represented as `[]` in Collatex JSON) in a column, indicating that the witness does not have a word at that alignment position.
* **Normalization Key (`n`)**: The comparative token string used under the hood by Collatex to determine matching and alignment (e.g., lemma, lemma+pos, or raw word).
* **Display Text (`t`)**: The original text token from the manuscript transcription. This is the text displayed on the Stemmaweb interface, regardless of which normalization key was used for the alignment.

---

## 1. Primary Graph-Theoretical Metrics (NetworkX-based)

These metrics analyze the mathematical topology of the **Variant Graph** directly, making them the most layout-independent, precise, and statistically relevant indicators of alignment quality.

### Nodes ($|V|$)
* **Definition**: The total number of unique nodes in the collation variant graph. It is a unique normalized display text (t) value within a single column.
* **Interpretation**: Fewer nodes indicate a more compressed and aligned graph. Note that variant readings (different transcribed texts, e.g. `λόγον` vs `λόγῳ`) must remain as separate parallel nodes to represent the variance. However, a better alignment strategy reduces the overall node count by:
  1. Standardizing surface-level differences (e.g. expanding abbreviations like `κς` $\rightarrow$ `κύριος` and stripping punctuation) so identical readings successfully merge into single nodes.
  2. Maintaining alignment synchronization, preventing CollateX from getting out-of-sync and creating duplicate separate nodes for identical words in the surrounding context.

### Edges ($|E|$)
* **Definition**: The total number of directed edges connecting the nodes in the graph.
* **Interpretation**: Represents the paths taken by different witnesses. Fewer edges generally denote less divergence and path complexity in the tradition.

### Merge Ratio
* **Definition**: The average number of witness tokens merged into each graph node.
* **Calculation**:
  $$\text{Merge Ratio} = \frac{N}{|V|}$$
  where $N$ is the sum of all tokens across all witnesses, and $|V|$ is the total number of nodes.
* **Interpretation**: This is the single most statistically robust metric for comparing alignment strategies because it normalizes input token count differences. A higher ratio indicates better compression and alignment.

### Variation Points (Divergence Nodes)
* **Definition**: Nodes in the graph with an out-degree greater than 1 ($out\_degree(v) > 1$).
* **Interpretation**: Represents branching points where the witnesses disagree on the following reading, creating a variant "bubble". Fewer variation points mean a cleaner, more linear graph with less text divergence.

## 2. Secondary Collation Matrix Metrics (Optional / Table-based)

> [!NOTE]
> **Optional / Secondary Status**: These metrics are calculated on the 2D alignment grid/table matrix layout projection. Because tables must introduce artificial **Gaps** (`-`) to represent alignment positions, these metrics are layout-dependent and less mathematically precise for evaluating underlying NLP alignment quality than the graph-based metrics.

### Total Columns
* **Definition**: The total length of the collation alignment matrix.
* **Interpretation**: A smaller number of columns generally correlates with a more collapsed graph.

### Consensus Columns
* **Definition**: The number of columns where all witnesses have a token and their normalization keys are 100% identical.
* **Calculation**:
  $$\text{Consensus} = \sum_{c \in \text{Columns}} [(\text{gaps}_c == 0) \land (|\{n_{w,c} \mid w \in \text{Witnesses}\}| == 1)]$$

### Variant Columns
* **Definition**: Columns where witnesses exhibit differences (including different normalization keys or gaps).
* **Calculation**:
  $$\text{Variant} = \sum_{c \in \text{Columns}} [|\{n_{w,c} \mid w \in \text{Witnesses} \land \text{cell}_{w,c} \neq \emptyset\}| > 1]$$

### Gap Columns
* **Definition**: The number of columns containing at least one empty cell.

### Total Gaps
* **Definition**: The total count of empty cells across the entire matrix.

### Alignment Density
* **Definition**: The proportion of non-empty cells relative to the total possible table cells.

---

## 3. Performance Metrics

### Tokenization Time
* **Definition**: The CPU time (in seconds) required to run the NLP parser (CLTK/Stanza), perform lemmatization and Part-of-Speech tagging, filter deleted tags, and format the tokens.

### Collation Time
* **Definition**: The HTTP time (in seconds) required to transmit payloads to the Collatex REST API and receive the aligned response.

### Total Time
* **Definition**: The overall execution time representing the sum of tokenization and collation phases.
